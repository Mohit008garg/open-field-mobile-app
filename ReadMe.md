# OpenField — Mobile App

React Native (Expo + expo-router) app for **OpenField**, a sports networking
platform — *Connect. Discover. Grow in Sports.* Built to the designs in
[`sample scrren/`](./sample%20scrren).

## Stack

- **Expo SDK 51** + **expo-router** (file-based routing), **TypeScript**
- Brand: vibrant green (`#16A34A`) on clean white/slate surfaces; card-based UI
- Icons via `@expo/vector-icons` (Ionicons)
- Responsive for phone + tablet

## Run

```bash
npm install
npm start          # Expo dev server (a = Android, i = iOS, w = web)
npm run typecheck  # tsc --noEmit
```

### Authentication — Google only (for now)

SMS OTP isn't purchased yet, so sign-in is **Google only**, using the **native**
Google SDK `@react-native-google-signin/google-signin` (OS account picker — no
browser/redirect). See [`src/hooks/useGoogleAuth.ts`](src/hooks/useGoogleAuth.ts).

> ⚠️ Native Google sign-in does **not** work in **Expo Go** — you need a dev
> build (`npx expo run:android`) or a standalone/EAS build.

**Setup**

1. `cp .env.example .env` and set `EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID` to your
   **Web** OAuth client ID (used to mint the ID token).
2. Create an **Android** OAuth client in the *same* Google project with:
   - package name `in.openfield.app`
   - the **SHA‑1** of the keystore that signs your build.
3. Run a dev build: `npx expo run:android`.

**Signing / SHA‑1 (avoids `DEVELOPER_ERROR`)**

`expo prebuild` generates a *random* `android/app/debug.keystore`. To keep a
stable SHA‑1 that matches the registered Android client, sign with your system
debug keystore:

```bash
cp ~/.android/debug.keystore android/app/debug.keystore
# SHA-1 to register in the Android OAuth client:
keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android | grep SHA1
```

Mismatched SHA‑1 → google-signin returns `DEVELOPER_ERROR`. For shared/CI builds,
use an EAS build profile with a fixed keystore (consistent SHA‑1).

`android/` and `ios/` are git-ignored (Continuous Native Generation) — they're
regenerated by `expo prebuild` / `expo run:*`.

A **dev "Skip"** link still enters the app. Phone (OTP) sign-in will be added
later.

### Backend session (JWT)

On a successful Google sign-in the app sends the Google **ID token** to the
backend (`POST /auth/google`), which verifies it, upserts the user and returns
our own **access + refresh tokens**. These are persisted with
`expo-secure-store` (keychain/keystore; `localStorage` on web) and attached as a
`Bearer` token to authenticated API calls, with transparent refresh on `401`.

- API layer: [`src/api/`](src/api) — `config` (base URL), `tokenStore`,
  `client` (fetch + auto-refresh), `auth` (`signInWithGoogle`, `logout`).
- Session state: [`src/context/AuthContext.tsx`](src/context/AuthContext.tsx)
  (`useAuth()` → `isAuthenticated`, `user`, `signInWithGoogle`, `signOut`).
- Backend base URL: `EXPO_PUBLIC_API_BASE_URL` (defaults to
  `http://10.0.2.2:3000` on the Android emulator, `http://localhost:3000`
  elsewhere). See `.env.example`.

## Structure

```
app/
├── _layout.tsx            # root Stack
├── index.tsx              # redirect → /start
├── (auth)/                # start, login
└── (app)/                 # _layout = bottom Tabs + the 7 screens above
src/
├── theme/                 # colors, spacing, radius, font sizes
├── components/ui/         # Logo, PrimaryButton, Avatar, TextField, Card, Chip, SectionHeader
└── constants/mockData.ts  # placeholder players, jobs, events, messages, etc.
```

All list content is placeholder data in `src/constants/mockData.ts`; swap for the
`Open-field-be` API when endpoints are ready. Avatars are initials-based (no
network); hero/cover banners use remote placeholder images.
